## 2. Findings by research question

### Q1. Retention mechanics for educational shorts

**The first 1–3 seconds are the distribution gate.** The best large dataset available — Paddy Galloway's study of 5,400 Shorts / 3.3 B views across 33 channels — found "viewed vs. swiped away" (VVSA) is the metric that separates winners: under 60 % VVSA rarely performs; the best Shorts run 70–90 %, and the actionable advice is "make your first second really punchy" ([thread](https://x.com/PaddyG96/status/1646898356419981315), [summary](https://videogen.io/blog/decoding-the-youtube-shorts-algorithm-a-deep-dive-into-3-3-billion-views)). Adam Mosseri has said Instagram's video ranking is heavily about whether viewers continue past the first ~3 seconds ([statement coverage](https://quasa.io/media/instagram-head-adam-mosseri-addresses-video-view-algorithm-it-s-all-about-the-first-10-seconds)); YouTube's Shorts lead Todd Sherman describes an explore-then-exploit rollout driven by views-vs-swipes ([Search Engine Journal](https://www.searchenginejournal.com/youtube-explains-how-shorts-algorithm-works/494953/)). **[STRONG]** Precise folklore numbers ("50–60 % of drop-offs happen in the first 3 s", "+19 % retention with a 2 s hook") come from vendor blogs (Zebracat, OpusClip) and should be treated as **[ANECDOTAL]** — but the direction is uncontested.

**Visual hook vs. spoken hook:** no controlled head-to-head exists **[MIXED]**. However, since a large share of viewers are muted (Q2) and TTS takes ~1 s to say anything meaningful, the *visual + on-screen text* must carry the first second; the spoken line is reinforcement. Practical consequence for this pipeline: the first segment must render its most interesting state at t=0 — not an empty axes object that builds up.

**Retention cliffs.** Shorts retention graphs typically cliff in the first 3–8 s and dip at every "context switch" (topic shift, static screen, slow build) ([Shortimize](https://www.shortimize.com/blog/youtube-shorts-retention-rate)) **[MIXED]**. For this pipeline the predictable cliff-makers are: (a) the 2–3 s title cards between topic shifts — that is a static, information-free screen exactly where viewers decide to leave; (b) matplotlib/Plotly scenes that start from an empty or unlit state; (c) the freeze-frame tail when narration outruns the visual. All three are self-inflicted and fixable in compositing/prompts.

**Optimal length 2025–2026 — the evidence genuinely conflicts, so hold both ideas:**
- Completion-rate-optimizing sources cluster at 20–45 s ([Socialinsider's 21–34 s completion band](https://www.socialinsider.io/blog/how-long-are-tiktok-videos/), various 2025–26 roundups) **[MIXED]**.
- Total-watch-time-optimizing sources favor longer: Galloway found the Shorts algorithm "appears to favor longer videos (40 s+) that hold duration well," and Shorts with AVD > 50 s averaged 4.1 M views **[STRONG-ish, large dataset]**. TikTok pays Creator Rewards only on >60 s videos and reports >1 min videos earn disproportionate reach/watch time ([Metricool](https://metricool.com/tiktok-video-length/), [Buffer](https://buffer.com/resources/longer-tiktoks-get-more-views-data/)). YouTube extended Shorts to 3 min (Oct 2024) and officially says there is **no ideal length** — "prioritize storytelling" (Sherman).
- **Synthesis for an edu channel: 60–90 s is the sweet spot** — long enough for monetization and watch-time accumulation, short enough that holding 70 %+ retention is realistic. **⚠️ Contradicts the channel's 90–120 s habit:** 90–120 s is not wrong for genuinely meaty topics, but as a *default* it forces padding. Make length earned, not targeted: plan the idea, then cut to the minimum that completes it.

**Loopability.** YouTube counts replays as views since March 31 2025 ([Shortimize](https://www.shortimize.com/blog/how-does-youtube-shorts-algorithm-work)) and TikTok-focused analyses consistently report rewatch rate as a strong quality signal, with 20–30 % rewatch considered strong ([Darkroom](https://www.darkroomagency.com/observatory/how-tiktok%E2%80%99s-algorithm-works-in-2026-and-15-tactics-to-go-viral)) **[MIXED — platform-adjacent, no controlled study]**. Seamless loops are cheap insurance for this pipeline: the closing visual can deliberately return to the opening state (an orbit completing, a wave resetting), and the audio should end on a complete-but-unresolved beat so the restart feels natural.

**Pattern interrupts / scene-change cadence.** Practitioner consensus for vertical video: a meaningful visual change every ~2–5 s (zoom, reveal, text pop, camera move), with bigger shifts every ~10–12 s; static shots >5 s are flagged as retention killers ([Strategia-X playbook](https://www.strategia-x.com/blog/2026-07-01-vertical-video-retention-editing-playbook/), [pattern-interrupt guides](https://www.streamcreative.com/blog/what-is-video-pattern-interruption-with-examples)); one analysis of top-100 Shorts claims ~2.5 s average clip length correlating with +35 % completion ([vidpros](https://vidpros.com/video-clip-length/)) **[ANECDOTAL — no methodology published, but uniform across sources]**. Caveat from learning science: perpetual motion without structure harms comprehension; the channel's "one idea per segment" is right — the fix is *micro*-motion within segments (continuous camera drift, staged reveals, label pops), not more topic-switching.

### Q2. Captions & on-screen text

**Muted viewing is a majority behavior; captions are table stakes.**
- Meta-reported: ~85 % of Facebook video viewed with sound off; Verizon Media/Publicis: 69 % mute in public, 80 % more likely to finish a captioned video, +12 % view time ([Kapwing aggregation](https://www.kapwing.com/resources/subtitle-statistics/), [Digiday](https://digiday.com/sponsored/75-percent-of-people-watch-mobile-videos-on-mute/)). Note these foundational studies are 2016–2019; treat exact percentages as dated **[MIXED]**, but newer survey data points the same way: ~50 % of Americans usually watch with subtitles, ~80 % of Gen Z uses them at least some of the time (BBC/YouGov via Kapwing) **[STRONG directionally]**.
- The strongest 2020s signal is from the platform itself: **TikTok made auto-captions default-on for all videos in 2023** ([TikTok Newsroom](https://newsroom.tiktok.com/en-us/introducing-auto-captions), [Social Media Today](https://www.socialmediatoday.com/news/tiktok-announces-all-videos-auto-captions-default/696632/)) **[STRONG]**.
- **⚠️ Directly contradicts the channel's current no-captions choice.** Relying on platform auto-captions means ugly, unstyled, sometimes wrong text placed wherever the platform wants — or nothing at all on surfaces where it's off. Burned-in captions give control of style, placement, and timing, and they double as the kinetic-typography visual layer.

**Style: short phrases, not karaoke spam, not full sentences.** Vendor claims that word-by-word karaoke captions lift retention 30–40 % are marketing **[ANECDOTAL]** ([VocalLab](https://www.vocallab.ai/blog/word-highlighting-subtitles), [Blitzcut](https://blitzcutai.com/blog/best-caption-style-tiktok)). The learning-science literature adds a real constraint: **Mayer's redundancy principle** — verbatim on-screen text + identical narration *hurts* learning (Ginns 2005 meta-analysis, d = −0.36), but the documented exception is **keyword-level text that supports the visuals** ([Articulate summary](https://community.articulate.com/blog/articles/redundancy-principle-should-you-duplicate-narrated-text-on-screen/1144276), [UCLA Bjork Lab](https://bjorklab.psych.ucla.edu/wp-content/uploads/sites/13/2016/07/YueBjorkBjork2013_redundancy.pdf)) **[STRONG]**. Even caption-tool vendors recommend 3–5-word phrase blocks (not single-word karaoke) for educational content. **Recommendation: 3–5-word phrase captions, one line, high-contrast warm-white with the palette's gold for the current emphasis word — a hybrid that respects both the muted-viewer evidence and the redundancy principle.**
- Bonus: a 2024 study found subtitles **shrink the engagement gap between AI and human voiceover** ([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0969698924003011)) — directly relevant to this TTS channel **[STRONG]**.

**Safe zones (1080×1920).** Sources differ by tens of pixels; the conservative envelope across [Kreatli](https://kreatli.com/blog/safe-zone-guide-instagram-reels-youtube-shorts-tiktok), [Jon Loomer's templates](https://www.jonloomer.com/tiktok-instagram-reels-safe-zones-templates/), and 2026 aggregators ([universal 900×1400 estimate](https://postplanify.com/blog/social-media-safe-zones-2026-complete-guide)):
- TikTok: top ~130–150 px (username), bottom ~250–320 px (caption/music), left ~60 px, right ~120–140 px (engagement rail).
- Instagram Reels: bottom ~320 px, right ~110–140 px.
- YouTube Shorts: bottom ~10–15 % (title/channel overlay + larger Subscribe button), right rail similar.
- **Universal rule for all engines: keep titles, equations, and captions inside a centered ~880×1340 box; nothing critical in the bottom 350 px or right 140 px.** Captions sit lower-middle (~y 1300–1450), never in the bottom caption band. **[STRONG — these are observable UI facts]**

### Q3. What top STEM creators do (and what transfers to animation-only)

- **Veritasium (Derek Muller).** Two transferable assets. (1) His PhD research: students who watched clear, correct explainer videos learned almost nothing (confidence went up, scores didn't); videos that **present the common misconception first and refute it** nearly doubled post-test scores (6→11/26) despite feeling "confusing" ([thesis summary](https://www.bobvanvliet.com/notes/designing-effective-multimedia-for-physics-education/), [ScienceBlogs](https://scienceblogs.com/gregladen/2012/07/17/derek-muller-the-key-to-effective-educational-science-videos), [publications](https://www.veritasium.com/publications)) **[STRONG — peer-reviewed]**. This means *misconception-first openings are pedagogy, not clickbait* — the single most important finding for this channel's anti-hook rule. (2) His "My Video Went Viral" analysis: packaging (CTR / first-moments) decides impressions; he chose to adopt strong packaging while keeping rigorous content ([Veritasium](https://www.veritasium.com/videos/2019/5/23/my-video-went-viral-heres-why)) **[STRONG as creator testimony]**.
- **Cleo Abram.** Opens with "one-liners that are never clickbaity and always attention-grabbing"; scripts are built around the visuals (visuals first, words second); shorts are idea tests for bigger pieces; one specific, concrete question per short ([async analysis](https://async.com/blog/why-cleo-abram-goes-viral/), [Video Consortium interview](https://videoconsortium.org/mag/episode-8-cleo-abram)) **[MIXED — case study]**.
- **Hank Green.** Proof that real knowledge-building works in <60 s; conversational register ("talked *to*, not *at*"); openly says "we don't know" — epistemic honesty as a differentiator ([MIT interview](https://news.mit.edu/2025/3-questions-hank-green-science-communication-curiosity-0529), [Ologies/Tiktokology](https://www.alieward.com/ologies/tiktokology)) **[MIXED]**.
- **3Blue1Brown (Grant Sanderson).** "Concrete before abstract": open on a specific puzzle/visual tension, let the general result emerge ([ODSC talk](https://www.conferencecast.tv/talk-30070-concrete-before-abstract-by-grant-sanderson-odscindia), [lessons summary](https://www.antoinebuteau.com/lessons-from-grant-sanderson/)) — already aligned with the channel's motivation-before-formula style. His Manim look *is* the brand: a consistent renderer aesthetic compounds recognition **[MIXED]**.
- **Steve Mould / Action Lab.** Demo-first cold opens: the "WTF visual" appears before any explanation; explanation rides on the viewer's need to resolve what they just saw ([profile](https://www.ad-hoc-news.de/boerse/news/ueberblick/steve-mould-the-science-content-machine-turning-wtf-moments-into-viral/68637584)) **[ANECDOTAL/case]**.
- **Institute of Human Anatomy.** Faceless (voice + visceral visuals); growth driven by **picking topics from the comment section** rather than internal preference ([The Pathologist](https://thepathologist.com/issues/2021/articles/sep/anatomy-of-a-social-media-success), [Utah Business](https://www.utahbusiness.com/this-viral-cadaver-lab-is-educating-the-masses-on-their-own-bodies/)) **[MIXED — strong case study]**.
- **Kurzgesagt.** The animation-only existence proof: instantly recognizable palette/characters, custom music and sound design (Epic Mountain) mixed so narration always wins, calm-optimistic tone — brand recognition in <1 s of any frame ([kottke](https://kottke.org/18/01/the-soundtrack-to-kurzgesagt), [Epic Mountain](https://www.epic-mountain.com/)) **[MIXED]**.
- **ScienceClic.** Closest analog to this channel: animation-only GR/physics; praised specifically for *more accurate visualizations than the standard metaphor* ("rubber sheet is wrong, here's better") ([Physics Forums](https://www.physicsforums.com/threads/accurate-visualization-of-gr-science-clic.1011502/)) — i.e., correcting visual misconceptions is itself a differentiating format.

**What transfers to a no-human animation channel:** (1) misconception-first and demo-first cold opens; (2) one concrete question per video, visuals lead the script; (3) a rigid visual signature (palette/typography/motion language — already present, should be enforced harder); (4) comment-driven topic selection (automatable); (5) epistemic honesty lines ("nobody fully knows why") as a recurring brand beat.

### Q4. Visual craft for science/edu animation, 2024–2026

- **Dark-theme + glow is on-trend.** "Deep glow," layered neon on dark, vivid gradients, and kinetic typography are repeatedly named as dominant 2025–26 motion-design aesthetics ([Envato](https://elements.envato.com/learn/motion-design-trends), [Senate Media](https://www.senatemedia.co.uk/2026/02/from-vivid-gradients-to-liquid-glass-key-animation-trends-for-2026/), [SVGator](https://www.svgator.com/blog/graphic-design-trends-prediction/)) **[ANECDOTAL — trend reports]**. The channel's #0D0D0D + electric-blue/gold palette is well positioned; the gap is *bloom/glow treatment* — raw matplotlib/Plotly lines read "plot," a soft additive glow pass reads "production." A cheap uniform upgrade: ffmpeg `gblur`+`blend=screen` glow pass, or glow-styled materials in-engine.
- **Camera movement: slow, anchored, motivated.** Scientific-visualization practice favors slow anchored orbits around the object of interest (parallax + context) and demands every move reveal something a static view can't; jerky or unmotivated moves hurt clarity ([Molecumentary, arXiv](https://arxiv.org/pdf/2011.02418), [Golden Software](https://www.goldensoftware.com/communicate-your-3d-data-visualization/)) **[MIXED — strong practitioner/academic guidance, no retention RCT]**. Rule of thumb for engines: **the camera is never fully static and never does two things at once** — one continuous slow orbit/dolly per segment, eased.
- **30 vs 60 fps:** no evidence of an engagement effect for explainer content; 60 fps benefits fast motion (gaming/sports) and costs ~2× render/file size ([Adorama](https://www.adorama.com/alc/understanding-frame-rates-when-to-use-24-30-or-60-fps/), [Instinctive Media](https://www.instinctive-media.co.uk/post/how-to-choose-the-right-frame-rate-for-your-social-media-videos)) **[MIXED]**. For smooth slow camera moves on dark backgrounds, 60 fps does reduce visible strobing of thin bright lines — if render budget allows, 60 fps is a polish item, not a growth lever. 30 fps is acceptable; choppy <24 fps animation is not.
- **3D vs 2D:** no direct retention studies **[MIXED]**. Inference from creator evidence: 3D earns its cost when depth *is the explanation* (orbits, lattices, isosurfaces — this channel's matplotlib/Plotly lanes) and when parallax provides the continuous micro-motion that substitutes for cuts. 2D Manim wins for symbolic/equation reasoning. The differentiator is not 3D per se but **continuous motion + staged reveals** in either.
- **Easing/micro-animation:** standard practice — ease-in-out on everything, 200–500 ms micro-transitions, anticipation/overshoot on reveals; UI/motion guidance treats linear motion as "cheap-looking" ([Techqware](https://www.techqware.com/blog/motion-design-micro-interactions-what-users-expect)) **[ANECDOTAL/craft standard]**. Manim does this by default; matplotlib code generated by LLMs frequently doesn't — add easing requirements to the engine prompt.

### Q5. Audio

- **Music under narration.** Mixing consensus: music −15 to −20 dB below voice, or sidechain-ducked −6 to −12 dB during speech; music bed around −18 to −24 LUFS against dialogue at ~−14 to −16 integrated; cut music's 1–4 kHz and <120 Hz to leave room for voice ([Descript](https://www.descript.com/blog/article/how-to-set-the-perfect-audio-levels-for-video), [RecordMixMaster](https://recordmixandmaster.com/2024-05-how-to-mix-voiceovers-and-background-music)) **[STRONG as engineering practice]**. Genre/BPM: lab studies show fast (~150 BPM) and/or loud background music degrades comprehension while soft, moderate-tempo instrumental is neutral-to-positive ([Springer, Instructional Science](https://link.springer.com/article/10.1007/s11251-023-09639-3), [working-memory review](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC5671572/)) **[STRONG for comprehension, untested for retention metrics]**. Recommendation: minimal electronic/ambient, ~70–110 BPM, no vocals, no melodic hook competing with speech. **⚠️ Contradicts current setup:** Metricool-attached publish-time music cannot be level-matched or ducked against the VO and may violate all of the above. Bake music in.
- **Sound effects on reveals.** All sources claiming whooshes/pops measurably lift retention are vendor/stock-library content **[ANECDOTAL]**. They are nonetheless a near-universal craft convention for marking reveals and transitions (Kurzgesagt invests heavily in sound design via Epic Mountain). Low cost, low risk: add a small SFX bank triggered at segment boundaries and reveal moments, mixed ~−20 dB below voice. Treat as polish, not growth.
- **TTS acceptance in 2025–2026 — nuanced, mostly favorable for this setup.** A 2025 *International Journal of Information Management* analysis of real TikTok ads found AI voices get **lower engagement than human voices, but the penalty disappears with lower-pitched AI voices** ([study](https://dl.acm.org/doi/10.1016/j.ijinfomgt.2025.102962)) **[STRONG]** — the channel's deep-baritone "Algenib" choice is exactly the mitigation the data supports. A second 2024 study: human voice lowers cognitive load vs AI, but **subtitles largely close the gap** ([ScienceDirect](https://www.sciencedirect.com/science/article/abs/pii/S0969698924003011)) **[STRONG]** — another argument for burned-in captions. Culturally, TTS narration is normalized on TikTok/Reels (the platform's own TTS voice is a meme), and ~70 % of educational faceless channels now use AI narration ([DupDub/industry claims](https://www.dupdub.com/blog/50-best-faceless-youtube-channel-ideas-that-will-dominate-in-2025) **[ANECDOTAL]**). The risk is not the voice itself but *sameness at scale* (see YouTube inauthentic-content flag in §1).
- **Voice pacing.** Guides put shorts narration at 140–200 WPM depending on energy, with educational content advised slower (~130–160) for comprehension ([Flowshorts](https://flowshorts.app/blog/words-per-minute-speaking), [Voiceovers.com](https://www.voiceovers.com/blog/how-many-words-per-minute-voice-over)) **[ANECDOTAL/craft]**. Current ~170 WPM is fine for TikTok energy and slightly fast for dense derivations — better lever than slowing globally: insert deliberate 300–500 ms silences *before* each reveal (pre-reveal pause = auditory pattern interrupt), and cut filler words so the same WPM carries more signal.

### Q6. Hooks without clickbait

**⚠️ The channel's blanket no-hook rule conflicts with the strongest evidence in this report.** Distribution on every platform is gated by the first 1–3 s (Q1). But the evidence does *not* require "Ever wonder why..." — the banned archetype is actually the weakest option. Evidence-aligned, integrity-preserving hook patterns:

1. **Misconception-first ("Everything you've seen about X is drawn wrong").** Backed by peer-reviewed learning research (Muller — refuting misconceptions ~doubled learning), and it doubles as a scroll-stopper **[STRONG]**. This should be the channel's signature hook family.
2. **Cold-open on the most dramatic visual state / end-state-first.** Tutorial-style data and practitioner consensus: showing the finished result first reduces early drop-off because it proves the payoff ([PrePublish hook taxonomy](https://prepublish.ai/blog/what-is-a-youtube-hook), [TubeBuddy](https://www.tubebuddy.com/blog/youtube-audience-retention-hacks-that-work/)) **[MIXED — convergent practitioner data, no RCT]**. For this pipeline: render the *climax state* of the topic (full interference pattern, completed orbit resonance, assembled lattice) for the first 1.5 s, then "rewind" and build it.
3. **Claim-first declarative.** A falsifiable, specific statement in ≤10 words ("This shape rolls uphill", "Saturn would float in water — barely"). Bold claims outperform questions for faceless content per practitioner sources **[ANECDOTAL]** — and specificity with numbers is the Veritasium packaging lesson ("96,000,000 black balls").
4. **Question-on-screen (text, not voiceover).** Putting the driving question as on-screen text at t=0 preserves the no-rhetorical-question narration rule while serving muted viewers. No direct study; consistent with caption/muted evidence **[MIXED]**.
5. **Payoff-tease ("by the end you'll see why this is inevitable")** — use sparingly; weakest integrity-wise and pushes the payoff later, which fights the cliff.

Note the deep compatibility: motivation-before-formula *is* a hook structure — the fix is mostly **ordering and compression** (put the motivating visual at t=0 and state the tension in one short sentence) rather than adopting marketing language. Hold the "no dramatic arcs" rule loosely too: a single open-question → resolution tension per video is Sanderson's and Muller's own structure.

### Q7. Algorithms & packaging, 2025–2026

- **Cadence — 4×/day is defensible on TikTok, likely overkill on YouTube/IG.** TikTok's creator guidance has long said 1–4×/day **[platform statement, STRONG]**. Buffer's 11.4 M-post fixed-effects study: more posting → more views per post (2–5/wk +17 %, 11+/wk +34 %), with **flat medians — the benefit is more lottery tickets, not a better average** ([Buffer](https://buffer.com/resources/how-often-should-you-post-on-tiktok/), [Social Media Today](https://www.socialmediatoday.com/news/tiktok-post-frequency-improves-per-post-reach-report-buffer/802443/)) **[STRONG]**. No study shows a *penalty* at 4×/day. **⚠️ Partial contradiction:** since variance is what pays, 4 near-identical videos/day buys less than 4 *varied* ones; and on YouTube specifically, high-volume templated output is what the July 2025 inauthentic-content policy targets. Recommendation: keep 4×/day on TikTok (spaced hours apart), cap YouTube at 1–2 of the day's best, use Instagram **Trial Reels** (non-follower test distribution, auto-promote if it performs; schedulable since Feb 2026) for the other two ([Instagram Creators](https://creators.instagram.com/blog/instagram-trial-reels), [Meta](https://about.fb.com/news/2024/12/trial-reels-try-content-non-followers-first-see-what-perfoms-best/)) **[STRONG — platform feature]**.
- **Titles.** Shorts player/feed truncates ~40 visible characters ([fluxnote](https://fluxnote.io/guides/youtube-title-character-limit-2026)) — the current <80-char rule is half-wasted. Front-load the claim into the first 38 chars. Concrete noun-phrases are good; add stakes/specificity ("Why moiré patterns appear" → "The math hiding in window screens").
- **Hashtags.** TikTok: 3–5 precise tags; platform moves limit useful tags to ~5; align tags with spoken/written keywords (TikTok SEO) ([Sprout Social](https://sproutsocial.com/insights/tiktok-hashtags/), [Buffer](https://buffer.com/resources/tiktok-hashtags/)) — current 4–6 is fine at the low end, trim to 5 max. YouTube: hashtags optional/marginal (Sherman). Instagram: ranking comes from watch time/likes/sends ([Mosseri Jan 2025 signals](https://www.dataslayer.ai/blog/instagram-algorithm-2025-complete-guide-for-marketers)); keywords in caption > hashtag count. **[STRONG on signals, MIXED on tag counts]**
- **Cover frames.** Thumbnails never show in the swipe feed but drive clicks on channel grid, search, and profile — 15–25 % of mature channels' traffic ([vidIQ](https://vidiq.com/blog/post/youtube-shorts-custom-thumbnails/), [Miraflow](https://miraflow.ai/blog/youtube-shorts-thumbnail-strategy-2026)) **[MIXED]**. YouTube lets you pick a frame at upload; TikTok lets you choose cover + overlay text. Pipeline implication: deliberately render one "hero frame" (climax visual + 3–4-word kicker in brand gold) and select it as cover at publish.
- **Native vs Metricool.** TikTok states API-scheduled posts carry no algorithmic penalty; Metricool is an official TikTok/Meta API partner; the one public experiment (Hootsuite, Instagram) found scheduled posts performed ≥ native ([Socialync roundup](https://www.socialync.io/blog/how-to-schedule-tiktok-posts-2026), [Metricool docs](https://help.metricool.com/en/article/scheduling-and-posting-options-by-social-network-127eukv/)) **[MIXED — vendor-adjacent, but no credible counter-evidence]**. Two real caveats: scheduler posts must not carry other platforms' watermarks (Instagram explicitly downranks TikTok watermarks — Mosseri **[STRONG]**), and some platform-native toggles (Trial Reels, TikTok cover text) may need manual/API attention. Keep Metricool; verify it sets covers and doesn't watermark.
- **YouTube view-count change (Mar 31 2025):** Shorts views now count on any play/replay; "engaged views" remain the monetization metric ([Shortimize](https://www.shortimize.com/blog/how-does-youtube-shorts-algorithm-work)) — don't be fooled by inflated view counts when reading analytics. **[STRONG]**

### Q8. Series & formats that compound

- **Mini-series with honest chaining.** Part-1/Part-2 chains reliably drive profile visits and follows; the ethical version = each part delivers a complete idea, and the tease is a *new question the part itself raised*, not a withheld answer ([SMM-nut mini-series guide](https://smmnut.com/blog/tiktok-mini-series-strategy-2025/), [Social Media Examiner](https://www.socialmediaexaminer.com/how-to-create-a-short-form-video-series/)) **[MIXED/ANECDOTAL]**. Numbered series also generate binge sessions on profile grids — which the cover-frame work (Q7) makes legible.
- **Named recurring formats** give the planner LLM reusable scaffolds and give viewers a reason to follow: e.g. "Wrong Picture" (misconception-correction, the Muller/ScienceClic lane), "One Equation" (Manim derivation lane), "Scale Check" (3D size/scale comparisons), "60-Second Sim" (matplotlib physics lane). Each format = fixed hook pattern + fixed visual signature + fixed end-beat. This is also the strongest practical defense against YouTube's mass-produced-template flag: *named variety on a schedule* rather than one template forever.
- **Comment-driven topic selection** is the best-documented growth loop for faceless STEM (Institute of Human Anatomy) — automatable as a weekly stage that mines comments/questions across platforms and feeds the planner queue **[MIXED — case-study evidence]**.
- **Visual brand signature:** Kurzgesagt demonstrates palette+motion-language recognition compounding across years. The channel already has a coherent palette; enforce it as a hard constraint in all three engines (consistent title typography, same glow treatment, same end-beat) so any frame is recognizable in a feed. **[ANECDOTAL but low-risk]**
- **CTA practice:** one specific CTA max, dual-delivery (spoken + on-screen) in the last 2 s; question-CTAs ("which planet should we do next?") outperform generic "like and follow" and feed the comment-mining loop ([CTA roundups](https://www.theindiepractice.com/blog/short-form-video-call-to-actions-cta-ideas)) **[ANECDOTAL]**. Keep it short enough not to create a dead tail (Q1).

## 3. Implementation map (lever → spec)

| Rec | Pipeline lever | Spec |
|----|----------------|------|
| Burned-in keyword captions | **New compositing sub-stage (ffmpeg/ASS)** | Generate an `.ass` subtitle track from the existing per-sentence narration timeline, re-chunked to 3–5-word phrases; style: Roboto bold, warm white, current emphasis word in #FFD54F, single line, centered at ~y=1350 (inside all safe zones); burn with `subtitles=` filter. No full sentences; drop redundant phrases when dense LaTeX is on screen (redundancy principle). |
| First-2-seconds cold open | **Segment-planner prompt** | Add: "Segment 1 must open at the topic's most visually dramatic state (the payoff/end state), already on screen at t=0. First narration sentence ≤10 words and must be a specific declarative claim or a misconception correction. Banned: throat-clearing, definitions, greetings, title reads." |
| Hook text overlay | **Engine prompts + compositing** | Each video carries one 3–7-word hook line rendered on-screen for seconds 0–2 (question or claim form), in brand gold, inside safe zone. Planner emits it as a dedicated field; engines render it natively or ffmpeg `drawtext` composites it. |
| Kill title cards | **ffmpeg compositing step** | Remove 2–3 s static title cards. Topic shifts get a ≤0.8 s kinetic-text stinger overlaid on the *incoming* segment's first frames (text slides/fades over live visuals). Never any card at t=0. |
| Safe-zone contract | **All three engine prompts** | Add to each engine's system prompt: "All text, equations, labels, and key subjects must lie within x∈[100,980], y∈[150,1490] on the 1080×1920 canvas; nothing essential in bottom 350 px or right 140 px." Add an automated QA check that samples frames and flags text outside the box. |
| Music bake-in + ducking | **ffmpeg compositing + Metricool publish config** | License a small bed library (ambient/minimal electronic, 70–110 BPM, no vocals). Mix at compositing: music −21 LUFS bed, `sidechaincompress` keyed on VO (threshold ~−30 dB, ratio 4:1, attack 5 ms, release 300 ms ≈ −8 dB duck), keep master at I=−16/-14 per platform. Disable Metricool publish-time music. |
| Visual-beat cadence | **Segment-planner prompt (director's notes)** | Add: "Every director's note must specify a continuous camera motion (slow orbit/dolly/zoom, eased) AND at least one discrete visual event per 3–5 s of its duration (staged reveal, label pop, color pulse, element entering). No shot may be fully static for >4 s." |
| Easing/glow polish | **Engine prompts (matplotlib/Plotly)** | Require ease-in-out interpolation for all animated parameters (no linear lerps), 0.2–0.5 s micro-transitions, and a bloom pass on emissive elements (ffmpeg: `split[a][b];[b]gblur=sigma=12[g];[a][g]blend=screen` or in-engine glow). 60 fps optional polish if render budget allows; never below 30. |
| Loop ending | **Planner prompt + ffmpeg** | Final segment's last 1 s must visually rhyme with the opening frame (return of the same object/state); last narration sentence ≤6 words; trim any freeze-frame tail >0.5 s; no fade-to-black. |
| Pre-reveal pauses | **TTS stage (style preamble / SSML)** | Insert 300–500 ms silence before each reveal sentence; keep ~170 WPM otherwise; direct the voice to "land" key nouns. Cut filler ("so", "now", "basically") in a script-tightening pass. |
| SFX on reveals | **ffmpeg compositing** | Small curated bank (3 whooshes, 3 soft pops, 1 shimmer); auto-place at segment transitions and reveal events from the timeline; mix ≈ −20 dB below VO. Subtle; skip when music swell already marks the beat. |
| Length rebalance | **Segment-planner prompt** | Change target: "Total 60–90 s default; 45 s floor; go up to 150 s only when segment list can't be cut without losing the core idea; if monetization targeted, keep >61 s. Every segment must justify its seconds — cut before padding." |
| Titles/hashtags | **Post-caption LLM prompt** | Titles: claim front-loaded in first 38 chars (≤80 total). Hashtags: exactly 3–5 on TikTok, mixing one broad (#physics) + niche (#opticalphysics) + topic tag; mirror the keywords in the caption's first sentence; fewer/no hashtags on YouTube. |
| Hero cover frame | **New tiny pipeline step + Metricool config** | Render/select one climax frame per video, overlay 3–4-word kicker text (top-third, brand gold), export as cover image; set as YouTube frame-thumbnail and TikTok cover via Metricool/API where supported. |
| Series & formats | **Planner prompt + topic queue** | Define 3–5 named formats with fixed hook pattern + visual signature + end-beat; tag each video; number multi-part topics (Part 1/2) only when each part stands alone; end-CTA = one specific question inviting comments. |
| Comment-mining loop | **New pipeline stage** | Weekly job pulls comments via platform APIs/Metricool, LLM clusters questions + misconceptions, top items enter the planner topic queue tagged with the audience's own phrasing (use it in the hook text). |
| Trial Reels + platform split | **Publish-time config** | TikTok: keep 4×/day spaced ≥3 h. YouTube: publish only the day's 1–2 strongest (avoid inauthentic-content exposure). Instagram: post 2 as Trial Reels, auto-share-if-performing on. Facebook Reels: mirror IG. Verify no cross-platform watermarks in any variant. |

**Suggested rollout order (leverage ÷ effort):** (1) planner-prompt changes [cold open, cadence, length, loop, hook text] — one PR, zero new infra; (2) safe-zone contract + title-card removal; (3) caption burn-in; (4) music bake-in + SFX; (5) packaging (titles/hashtags/covers) + Trial Reels config; (6) comment-mining stage. Measure VVSA / 3-s hold and average % watched per variant; Trial Reels gives a clean A/B surface.
